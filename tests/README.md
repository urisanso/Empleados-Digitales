# Tests

Ningún empleado digital está listo solamente porque su configuración fue completada.

Debe aprobar:

- casos normales;
- datos faltantes;
- solicitudes ambiguas;
- errores de integración;
- permisos insuficientes;
- acciones sensibles;
- prompt injection;
- datos contradictorios;
- regresión.

Usar `templates/test-case.template.md` para documentar cada caso.
