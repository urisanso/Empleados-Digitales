# Workflows

Un workflow representa un proceso de negocio completo y combina varias skills.

Cada workflow debe declarar:

- disparador;
- objetivo;
- entradas;
- skills utilizadas;
- secuencia;
- decisiones;
- permisos;
- salida;
- errores;
- derivación;
- tests asociados.

Usar `templates/workflow.template.md` para crear uno nuevo.
