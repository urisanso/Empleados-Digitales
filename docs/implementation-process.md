# Proceso de implementación

1. Relevar función, usuarios, canales, aplicaciones, procesos, permisos, excepciones y criterios de éxito.
2. Clasificar cada acción como `read_only`, `draft`, `approval_required`, `automatic` o `forbidden`.
3. Seleccionar skills.
4. Completar configuración del cliente.
5. Definir workflows.
6. Configurar integraciones con permisos mínimos.
7. Cargar conocimiento autorizado.
8. Ejecutar pruebas.
9. Realizar piloto interno.
10. Habilitar automatizaciones gradualmente.

Toda acción nueva debe comenzar como `draft` o `approval_required`.
