# Arquitectura

## Core

Reglas obligatorias para todos los agentes.

## Clients

Configuración declarativa específica de cada implementación.

## Skills

Capacidades reutilizables con responsabilidad concreta.

## Workflows

Procesos de negocio que combinan skills.

## Integrations

Contratos y adaptadores para sistemas externos.

## Knowledge

Información consultable sin lógica operativa.

## Tests

Validación funcional, de permisos, seguridad y regresión.

## Separación obligatoria

No mezclar credenciales con documentación, datos del cliente con lógica genérica, workflows con conectores ni conocimiento comercial con permisos.
